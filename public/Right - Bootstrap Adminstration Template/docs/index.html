Right - Responsive Admin Template

Right is pretty simple and light Bootstrap template that will help you to build the admin panel for your project.

Version: 1.0.0

Documentation

These docs is a brief description of the key points that can be useful to you in the process of working with the template.

Overview

We develop The Right on the principles of combining beautiful and functional design. In addition to that, an important role is assigned to the speed of development and ease future code maintenance. So, The Right uses such tools as Pug (Jade) and Less for HTML templating and CSS-preprocessing. All source code is building automatically via Gulp. Together, they greatly simplify and speed up the development process. But don't worry if you have never worked with these tools. There is fully assembled and ready to go HTML pages included as well which you can immediately implement in your project.

Layout structure

All pages of the template (except login.html) consist of the following parts:

  1. .wrapper - main container for all elements
  2. .header-navbar - top navigation panel
  3. .dashboard - sidebar and content container
  4. .sidebar - side navigation panel
  5. .main - common content container
<div class="wrapper">
    <nav class="header-navbar"></nav>
    <div class="dashboard">
        <div class="sidebar"></div>
        <div class="main"></div>
    </div>
</div>

Responsivity

When the screen width becomes smaller than 768px, the template adapts. .sidebar is forced out of the screen and comes back again when you click the menu button. .header-navbar-mobile element replaces .topnavbar and contains the mobile menu button and the page title.

Pages

The template includes 8 functional pages:

  1. index.html - page with brief statistics and some informational widgets.
  2. products.html - the page can be used to manage the list of products or goods.
  3. orders.html - orders page.
  4. users.html - users or customers manage page.
  5. login.html - login page.
  6. blank.html - blank page, used to create new functional pages.
  7. inbox.html - mails list page.
  8. compose.html - new mail form.

Also there are some demo pages included in the template. They are used to demonstrate components of the interface.

Building the project

Automation assemblies tools are an important part of modern frontend development. If you don't use any of them, then we highly recommend you to explore such projects as Gulp, Grunt or Webpack. They will help you to automate many routine tasks and to improve the quality and speed of your work.

Getting started

First, you need to have the Node.js to be installed. If you don't have one, get it on nodejs.org.

Once Node.js is installed, go to the project folder, which contains your package.json file. Now we need to get the dependencies. Run the following command in OS command line:

npm install

This command will get the Gulp and all of its modules needed for further work.

If you get any error messages, make sure that you have the Node.js and npm installed correctly. And check the package.json file in the startup directory. Also, you can try to run the command as administrator.

Gulp

Gulp.js is the automated build system that we will use for generating html from pug and css from less. It also allows you to automate operations such as resource minification, files concatenation and more. There are most of necessary tasks prepeared in the gulpfile.js.

To begin building your project just write in the command line:

npm run build

This will start the builder that will process all the files in the ./source folder and will store the result in ./dist.

The command npm run watch will track all the changes in the folder ./source real-time and automatically update the result in ./dist.

To consistently run the task build and watch you can run npm run dev.

npm run clean deletes the directory ./dist.

Pug

Pug is a powerful template engine, which allows to considerably speed up and simplify the elaboration of html projects. This project was renamed from Jade. Read more about its capabilities and syntax on the official website.

The structure of pug templates:

Right/
  ├ source/                              source files
  ├───pug/                               pug folder
  ├──── include/                         static content files
  ├──── parts/                           common parts of template
  ├────── layout.pug                     common layout
  ├────── main.pug                       main block
  ├────── navbar.pug                     top navigation
  ├────── sidebar.pug                    side navigation
  ├──── blank.pug                        all end pages
         .
         .
         all *.pug pages

Any final page begins with the inheritance common template layout.pug , which generaly looks like this:

doctype html
html(lang="en")
    head
        meta(charset='utf-8')
        meta(http-equiv='X-UA-Compatible', content='IE=edge')
        meta(name='viewport', content='width=device-width, initial-scale=1')
        title Right - Bootstrap Admin Template
        block css
        link(href='css/right.css', rel='stylesheet')
    block vars
    body
        .wrapper
            include ./navbar.pug
            .dashboard
                include ./sidebar.pug
                include ./main.pug
        script(src='libs/jquery/jquery.min.js')
        script(src='libs/bootstrap/js/bootstrap.min.js')
        block js
        script(src='js/main.js')

Each page is constructed on the basis of this layout, and includes individual parts using the operator include, replacing the special tags block with they own content.

Any final page template looks like (for example, blank.pug ):

extends ./parts/layout.pug
block vars
    - page = 'Blank';
block main
    .main-heading
        .main-title
            ol.breadcrumb
                li.active Blank
        .main-filter
            form.main-filter__search
                .input-group
                    input.form-control(type="text", placeholder="Search...")
                    span.input-group-btn
                        button.btn.btn-default(type="button")
                            .fa.fa-search
    .container-fluid.half-padding
        .template.template__blank
            .row
                .col-md-12
                    .panel.panel-danger
                        .panel-heading
                            h3.panel-title Blank Page
                        .panel-body
                            p This is a blank page.

Less

Less is a CSS pre-processor. It extends the CSS language, adding such features as variables, mixins, functions and many other techniques. It allow you to make maintainable, themable and extendable CSS.

The structure of less directory:

Right/
  ├ source/                              source files
  ├───less/                              less folder
  ├──── dark/                            color theme container folder
  ├────── blocks/                        layout structure elements
  ├────── bootstrap/                     customized bootstrap styles
  ├────── libs/                          customized third-party libs
  ├────── colors.less                    colors
  ├────── mixins.less                    less mixins
  ├──── right.dark.less                  color theme container file

Color Themes

Version 1.0.0 includes 3 color schemes: dark, lilac and light. The styles for each of them are located in separate files. Thus, to use a specific color scheme, you need to connect to the page a certain stylesheet.

Right/
  ├ dist/
  ├─── css/
  ├───── right.dark.css             Dark theme
  ├───── right.light.css            Light theme
  ├───── right.lilac.css            Lilac theme
