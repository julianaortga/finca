Right - Responsive Admin Template

Right это довольно простой и легкий Bootstrap шаблон для административной панели вашего проекта.

Версия: 1.0.0

Документация

Далее располагается краткое описание ключевых моментов, которые могут помочь вам в процессе работы с шаблоном.

Обзор

Помимо сочетания приятного и функционального дизайна, в основу Right заложены принципы ускорения процесса разработки и простоты дальнейщей поддержки проекта. Для этих целей в нем применяются такие инструменты как Pug (Jade) шаблонизатор и CSS-препроцессор Less. Весь исходный код собирается автоматически с помощью Gulp. Таким образом процесс разработки значительно ускоряется. Но применение этого инструментария не является обязательным условием использования Right в вашем проекте. Все исходные файлы уже собраны, включены в шаблон и подготовлены для вас.

Структура Шаблона

Все страницы (за исключением login.html) состоят из следующих частей:

  1. .wrapper - общий контейнер всех элементов
  2. .header-navbar - панел верхней навигации
  3. .dashboard - контейнер левого меню и контента
  4. .sidebar - левое навигационное меню
  5. .main - общий контейнер содержания страницы
<div class="wrapper">
    <nav class="header-navbar"></nav>
    <div class="dashboard">
        <div class="sidebar"></div>
        <div class="main"></div>
    </div>
</div>

Адаптивность

Когда ширина страницы становится меньше чем 768px, шаблон адаптируется. .sidebar уходит за рамки страницы и появляется снова после нажатия кнопки меню. .header-navbar-mobile замещает .topnavbar и содержит кнопку вызова меню и заголовок страницы.

Функциональные Страницы

Шаблон включает 8 функциональных страниц:

  1. index.html - страница с общей статистикой и информационными виджетами.
  2. products.html - страница для управления списком товаров.
  3. orders.html - страница заказов.
  4. users.html - страница управления пользователями.
  5. login.html - страница авторизации.
  6. blank.html - пустая страница, служит основой для создания новых.
  7. inbox.html - страница сообщений.
  8. compose.html - форма нового сообщения.

Помимо этого в шаблоне присутствуют некоторые демонстрационные страницы, которые служат для описания отдельных компонентов интерфейса.

Сборка Проекта

Автоматическая сборка проекта играет важную роль в современной разработке. Если вы не используете ее в своей работе, тогда мы настоятельно рекоментдуем вам ознакомиться с такими проектами, как Gulp, Grunt или Webpack. Они помогут вам избежать большого количества рутинной работы и повысят качество и скорость вашей разработки.

Начало работы

Для начала, вам нужно иметь установленный Node.js. Если у вас его еще нет, то вы можете взять его с официального сайта nodejs.org.

После того, как Node.js установлен, нужно перейти в корневую папку шаблона, где лежит файл package.json. Далее необходимо загрузить зависимости. Запустите следующую команду в вашей командной строке:

npm install

Таким путем мы установим Gulp и необходимые модули для него.

Если в проецессе установки возникают какие-либо ошибки, то убедитесь, что Node.js и npm установлены корректно, и вы находитесь в нужной директории. Также, можно попробовать запустить установку от имени администратора.

Gulp

Gulp.js это система автоматческой сборки, которую мы будем использовать для генерации HTML-страниц из pug-шаблонов, и CSS-стилей из LESS-файлов. Она также позволяет наладить процесс сжатия и минификации файлов, оптимизации изображений и многое другое. Большинство необходимых команд для этих задач уже определено в файле gulpfile.js.

Для запуска сборки проекта используется следующая команда:

npm run build

Это запустит сборщик, который обработает все файлы в папке ./source и сохранит результат в папку ./dist.

Команда npm run watch позволит обрабатывать все изменения файлов из ./source в режиме реального времени и автоматически обновлять содержимое ./dist.

Чтобы последовательно запустить задачи build and watch можно использовать команду npm run dev.

npm run clean удаляет содержимое ./dist.

Pug

Pug это мощный html-шаблонизатор, котрый позволяет одновременно упростить и ускорить процесс разработки и поддержки html-проектов. Это инструмент ранее был переименован из Jade. Подробнее ознакомиться с его возможностями и синтаксисом вы можете на его официальном сайте.

Структура pug-шаблонов:

Right/
  ├ source/                              исходные файлы
  ├───pug/                               pug директория
  ├──── include/                         файлы со статическим содержимым
  ├──── parts/                           общие части страниц
  ├────── layout.pug                     общий шаблон
  ├────── main.pug                       основной блок страницы
  ├────── navbar.pug                     верхняя навигация
  ├────── sidebar.pug                    левое навигационное меню
  ├──── blank.pug                        пример итоговой страницы
         .
         .
         все итоговые *.pug страницы

Любая итоговая страница начинается с наследования общего шаблона layout.pug , котрый в общих чертах выглядит следующим образом:

doctype html
html(lang="en")
    head
        meta(charset='utf-8')
        meta(http-equiv='X-UA-Compatible', content='IE=edge')
        meta(name='viewport', content='width=device-width, initial-scale=1')
        title Right - Bootstrap Admin Template
        block css
        link(href='css/right.css', rel='stylesheet')
    block vars
    body
        .wrapper
            include ./navbar.pug
            .dashboard
                include ./sidebar.pug
                include ./main.pug
        script(src='libs/jquery/jquery.min.js')
        script(src='libs/bootstrap/js/bootstrap.min.js')
        block js
        script(src='js/main.js')

Каждая страница строится на основе этого шаблона, подключая отдельные части через оператор include, и заменя особые тэги block на собственное содержимое.

Все итоговые страницы выглядят следующим образом (на примере blank.pug ):

extends ./parts/layout.pug
block vars
    - page = 'Blank';
block main
    .main-heading
        .main-title
            ol.breadcrumb
                li.active Blank
        .main-filter
            form.main-filter__search
                .input-group
                    input.form-control(type="text", placeholder="Search...")
                    span.input-group-btn
                        button.btn.btn-default(type="button")
                            .fa.fa-search
    .container-fluid.half-padding
        .template.template__blank
            .row
                .col-md-12
                    .panel.panel-danger
                        .panel-heading
                            h3.panel-title Blank Page
                        .panel-body
                            p This is a blank page.

Less

Less это CSS пре-процессор. Он расширяет стандартные возможности CSS, добавляя такие возможности, как переменные, примеси, функции и много другое. Он позволяет создавать более поддерживаемый, управляемый и расширяемый CSS.

Структура less директории:

Right/
  ├ source/                              исходные файлы
  ├───less/                              less директория
  ├──── dark/                            контейнер цветовой схемы
  ├────── blocks/                        элементы структуры шаблона
  ├────── bootstrap/                     измененный bootstrap стили
  ├────── libs/                          измененные сторонние библиотеки
  ├────── colors.less                    цвета
  ├────── mixins.less                    less примеси
  ├──── right.dark.less                  общий файл цветовой схемы

Цветовые Схемы

Версия 1.0.0 содержит 3 цветовые схемы: dark, lilac и light. Стили для каждой из них находятся в отдельном файле. Поэтому, для использования определенной цветовой схемы, вам нужно подключить на страницу соответствующий файл стилей.

Right/
  ├ dist/
  ├─── css/
  ├───── right.dark.css             Dark схема
  ├───── right.light.css            Light схема
  ├───── right.lilac.css            Lilac схема
